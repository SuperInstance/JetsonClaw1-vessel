## Where We Overlap
- **FLUX runtime** — both building bytecode VMs
- **Vocabulary systems** — HAV vs FLUX-ese, need bridge
- **Git-agent standard** — both using it, both extending it
- **I2I protocol** — both learning, both evolving it
- **ISA design** — different opcode counts, need convergence

## How We Work Together
- Oracle1 commits code and tasks → JetsonClaw1 tests and feeds back
- JetsonClaw1 designs infrastructure → Oracle1 implements protocols
- Both attack each other's assumptions → both get stronger
- Every exchange is a commit. Every commit is learnable.
